What happens when you initialize a repository? Why do you need to do it?

You create a .git directory. To track different versions of the file.




How is the staging area different from the working directory and the repository? What value do you think it offers?

The staging area contains the files that will be committed to the repository.
The working directory could contain additional files.




How can you use the staging area to make sure you have one commit per logical 
change?


You can check which files have changed since last commit.




What are some situations when branches would be helpful in keeping your history
organized? How would branches help?




When you want to add new features. Branches help you keep track of your changes.



How do the diagrams help you visualize the branch structure?

Drawing the branch makes it easier to follow where each branch leads.




What is the result of merging two branches together? Why do we represent it in
the diagram the way we do?


It combines the changes made in both branches. We represent it like this to make it easy to follow.



What are the pros and cons of Git’s automatic merging vs. always doing merges
manually?

Pros: When it automatically merges it save you time.
Cons: You might not be able to resolve it, so you will have to merge it manually.
